1. Введение
Цель работы — реализовать интерактивный демонстратор трёх ключевых структур данных: стека, очереди и двоичного дерева поиска (BST). Приложение должно предоставлять пошаговую визуализацию операций, журнал действий и возможность отмены последних операций (undo). В дополнение реализован веб-интерфейс на Flask, позволяющий последовательно просматривать состояние структур и выполнять операции через браузер.

2. Постановка задачи
Требуется создать программный продукт, реализующий:
- операции push/pop для стека;
- enqueue/dequeue для очереди;
- insert/delete/search для BST;
- пошаговую визуализацию состояний и журнал операций с возможностью экспорта;
- возможность отмены операций;
- простую веб-страницу (Flask) с навигацией по шагам (previous/next) и визуализацией BST в виде уровней ASCII.

3. Требования и ограничения
- Язык: Python (студент выбирает самостоятельно);
- Хранение данных: файловая система (JSON) — в данной реализации последовательность операций может быть загружена/сохранена в JSON-файлах, но основное состояние хранится в оперативной памяти;
- Контейнеризация: Dockerfile включён;
- Тестирование: реализованы pytest тесты (юнит и интеграционные);
- Отчёт 20–25 страниц — прилагается.

4. Выбор технологии и обоснование
Выбран Python по следующим причинам:
- широкий набор библиотек для быстрой разработки и тестирования;
- простота реализации алгоритмов и визуализации в ASCII;
- наличие лёгких веб-фреймворков (Flask) для создания демонстрационного интерфейса;
- простота контейнеризации с использованием официального образа Python.

5. Архитектура и структура проекта
Проект организован в несколько модулей:
- data_structures.py — основные структуры и менеджер состояний;
- visualize.py — (ранее) функции визуализации; обновлённый функционал включён в data_structures для сериализации;
- app.py / flask_app.py — CLI и веб-интерфейс;
- templates/, static/ — веб-ресурсы (HTML/CSS);
- tests/ — тесты (pytest);
- Dockerfile, requirements.txt, README.md.

6. Описание реализаций
6.1 Stack
Реализован как список Python (LIFO). Операции push и pop сохраняют снимок состояния до изменения и записывают операцию в журнал.

6.2 Queue
Реализована с использованием collections.deque (FIFO). Аналогично стеку, операции сохраняют снимки и логируют действия.

6.3 BST
Дерево реализовано посредством узлов (BSTNode) с полями left/right. Вставка, удаление (с учётом трёх случаев: лист, один ребёнок, два ребёнка с заменой на inorder-successor) и поиск реализованы рекурсивно. Для сериализации используется preorder с None-маркерами, что позволяет сохранять/восстанавливать структуру дерева.

6.4 История операций и undo
Менеджер сохраняет snapshot (копию структуры) перед каждой операцией в список history. При undo последний snapshot применяется назад, а запись операции убирается из журнала.

7. Веб-интерфейс (Flask)
7.1 UI/UX
Создан лёгкий интерфейс с формой для выполнения операций и кнопками навигации по шагам (предыдущий/следующий/сброс). Текущее состояние отображается в трёх панелях: стек, очередь и BST (уровни ASCII).

7.2 Навигация по шагам
На сервере поддерживаются списки snapshots и logs_per_snapshot. После каждой операции в web-режиме к списку добавляется текущее состояние; при навигации назад/вперёд сервер восстанавливает состояние менеджера по выбранному снимку и отображает соответствующий журнал.

7.3 Серверная генерация ASCII-визуализации
Для BST реализована визуализация по уровням (BFS). Узлы без детей обозначаются точкой "." при необходимости выравнивания. Для небольших деревьев вывод обеспечивает наглядное представление структуры.

8. Тестирование
8.1 Юнит-тесты
В tests/ присутствуют тесты для проверки корректности операций стека, очереди и BST, а также undo. Тесты запускаются с помощью pytest.

8.2 Интеграционные тесты
Тесты проверяют совместную работу менеджера состояний при последовательных операциях и восстановлении состояний.

8.3 Набор тестовых данных
sample_operations.json содержит пример последовательности операций, используемый в демонстрации и в качестве тестовых данных.

9. Docker-контейнеризация
Dockerfile позволяет собрать образ на базе python:3.11-slim, установить зависимости и запустить Flask-приложение.

10. Примеры запуска и сценарии использования
- Локально: python flask_app.py и открыть http://localhost:5000
- В Docker: docker build -t a04-demo . && docker run -p 5000:5000 a04-demo

11. Производительность и ограничения
- Дерево и операции реализованы в наивной форме, без балансировки (AVL/Red-Black), поэтому худший случай — O(n) при вставке отсортированных данных.
- Веб-демо хранит состояние в оперативной памяти; для очень больших данных потребуется иная стратегия хранения (файлы/БД).

12. Безопасность и дополнительные замечания
- Приложение предназначено для локального использования и обучения; не адаптировано для многопользовательской среды.
- Не проводились работы по ограничению пользовательских вводов (например, XSS в шаблонах) — в учебной среде это допустимо, но для реального продакшна необходимо дополнительно обработать ввод.

13. Заключение
Реализован функциональный интерактивный демонстратор структур данных с веб-интерфейсом и возможностью пошаговой навигации. Проект удовлетворяет требованиям задания: сохранение логов, возможность undo, тесты, Dockerfile и отчёт.
